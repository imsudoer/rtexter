# RTexter

> ⚠️ **README GENERATED BY AI**

A cross-platform desktop automation tool for scripted keyboard input and macro recording. Written in Rust using [`egui`](https://github.com/emilk/egui) for the GUI, [`enigo`](https://github.com/enigo-rs/enigo) for keyboard simulation, and [`rdev`](https://github.com/Narsil/rdev) for global hotkeys and macro capture.

This is a Rust rewrite of the outdated Python `HTexter` (tkinter-based), with a redesigned UI and extended functionality.

---

## Features

- **Script-driven text input** — type multi-line text into any focused window with configurable per-character and per-line delays.
- **Directive language** — embed control commands directly in your script (delays, key presses, custom typing, pauses).
- **Auto-repeat** — loop the entire script indefinitely until stopped.
- **Global hotkeys** — start/stop execution from anywhere, even when the app is not focused (default: `F6` / `F7`).
- **Macro recorder** — capture raw keyboard events with precise timing, save/load to JSON, replay at adjustable speed.
- **Script file I/O** — open and save `.txt` / `.htx` scripts.
- **Dark themed GUI** — collapsible settings panels, monospace editor, live progress bar.
- **Layout-independent shortcuts** — `Ctrl+A` / `Ctrl+C` / `Ctrl+V` / `Ctrl+X` work in the editor regardless of the active keyboard layout (Russian, etc.).

## Installation

### Prerequisites

- Rust 1.74 or newer (`rustup` recommended).

### Platform-specific dependencies

**Linux** (Debian/Ubuntu):
```bash
sudo apt install libxdo-dev libxtst-dev libx11-dev libxi-dev pkg-config
```

> **Note:** `rdev` requires an X11 session. Wayland is **not supported** for global hotkeys or macro recording. Check with `echo $XDG_SESSION_TYPE`.

**macOS:**
- Grant the binary (or your terminal/IDE during development) permission under:
  `System Settings → Privacy & Security → Input Monitoring`
  and `Accessibility`.

**Windows:**
- No additional setup required.

### Build & run

```bash
git clone https://github.com/imsudoer/rtexter.git
cd htexter-rs
cargo run --release
```

## Usage

### Quick start

1. Launch the app.
2. Type or paste your script into the central editor.
3. Adjust timings on the left panel (start delay, line delay, per-character delay).
4. Click **▶ Запуск** (Run) or press the global hotkey `F6`.
5. Switch to the target window within the start-delay window — input will be typed there.
6. Press `F7` at any moment to stop.

### Script syntax

Every non-empty line is typed as text. Lines starting with `#` are **directives**.

| Directive | Description |
|---|---|
| `# MANUAL_SLEEP <ms>` | Wait `<ms>` milliseconds before starting (overrides start delay if placed first). |
| `# MANUAL_DELAY <ms>` | Change inter-line delay mid-script. |
| `# MANUAL_WRITE_DELAY <ms>` | Change per-character delay mid-script. |
| `# key <name> [click\|press\|release]` | Send a key event. Defaults to `click`. |
| `# type "<text>" <ms>` | Type arbitrary text with a custom per-character delay. |
| `# pause <ms>` | Pause execution. |

Supported key names: `enter`, `tab`, `space`, `esc`, `backspace`, `delete`, `up`, `down`, `left`, `right`, `home`, `end`, `pageup`, `pagedown`, `shift`, `ctrl`, `alt`, `f1`–`f12`, and any single character (`a`–`z`, `0`–`9`).

### Example script

```
# MANUAL_SLEEP 2000
Hello, this is line one.
# key tab
Hello, this is line two.
# pause 500
# type "Inserted at custom speed" 15
# key enter
Done.
```

### Behavior options

- **Press Enter before/after each line** — when enabled, the script wraps each typed line in `Enter` keystrokes (useful for chat windows). Disable for plain text editors.
- **Auto-repeat** — restarts the script from the top after the last line.

## Macros

The macro recorder operates independently from the script engine.

1. Click **⏺ Record**.
2. Perform any keyboard actions you want to capture.
3. Press `F8` to stop recording (configurable in source).
4. Click **▶ Play** to replay.
5. Adjust playback **speed** between 0.1× and 5×.
6. Use **💾 Save** / **📂 Load** to save/load macros as JSON.

Macros store full press/release events with microsecond-precision timestamps.

## Global hotkeys

| Hotkey | Action |
|---|---|
| `F6` | Start script execution |
| `F7` | Stop script execution |
| `F8` | Stop macro recording (when recording is active) |

Hotkeys are handled by a single global `rdev` listener. If you see a red warning bar at the top of the window saying *"Глобальные хоткеи не работают"*, check the platform notes above.

## Architecture

```
src/
├── main.rs       # eframe entry point, window setup
├── app.rs        # egui UI, state, global listener wiring
├── engine.rs     # Script parser + execution engine (runs in worker thread)
└── macros.rs     # Macro recorder/player, JSON serialization
```

- **Script execution** runs in a dedicated `std::thread`, communicating with the UI via `Arc<AtomicBool>` (stop flag) and `Arc<Mutex<(usize, usize)>>` (progress).
- **Global input listening** uses a single `rdev::listen` thread that fans out events to both the hotkey handler and the macro recorder.
- **Keyboard simulation** uses `enigo` for script playback (Unicode text support) and `rdev::simulate` for macro replay (exact key events).

## Known limitations

- **Wayland (Linux)** is not supported for global hotkeys/macro recording — use X11.
- `enigo`'s `text()` typing may not work in some games or applications that bypass standard input APIs.
- Macro recording captures **keyboard only**; mouse events are not recorded.
- Key-name parsing for macros is limited to common keys (full list in `macros.rs::string_to_key`).
- The first attempt to install a global listener may fail silently on Windows if another low-level keyboard hook holds priority — restart the app.

## Configuration

Tweak defaults at the top of `app.rs`:

```rust
start_delay_ms: 3000,
line_delay_ms: 100,
write_delay_ms: 20,
hotkey_start: "F6",
hotkey_stop: "F7",
```

The stop key for macro recording is set in `macros.rs::MacroRecorder::new` (`stop_key: "F8"`).
